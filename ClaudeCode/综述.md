# Claude Code 综述

## 一、定义

Claude Code 是一个 **Agent 框架**。

### 1.1 公式

```
Claude Code = 引擎 + 工具 + 接口 + 管理

引擎 = LLM + AgenticLoop
AgenticLoop = 输入 → 推理 → 执行工具 → 加入执行结果 → 再推理
接口 = 记忆 + 能力 + 委派 + 事件 + 环境 + 编程
管理 = 状态 + 安全 + 上下文
```

### 1.2 架构

| 类型 | 组件 | 内容 | 本质 | 解决主要问题 |
|---|---|---|---|---|
| **引擎** | LLM | default/opus/sonnet/haiku | 推理引擎 | 理解意图与决策 |
| | AgenticLoop | 循环驱动 | 执行引擎 | 让系统运转起来 |
| **工具** | 工具 | Read/Edit/Write/Glob/Grep/Bash... | 操作文件、搜索、网络… | 动手操作 |
| **接口** | 记忆接口 | CLAUDE.md/Auto Memory | 跨会话持久化规则和知识 | 规则和知识的遗忘 |
| | 能力接口 | skills | 技能规范化 | 不按套路出牌 |
| | 委派接口 | subagents | 任务拆分与委派 | 上下文溢出 |
| | 事件接口 | hooks | 会话、工具等生命周期事件触发自定义逻辑 | 安全防控与善后处理 |
| | 环境接口 | MCP | 连接外部服务与数据 | 利用外部功能和资源 |
| | 编程接口 | Python SDK/TypeScript SDK/Headless | 程序化编排 | 将Agent能力融入到应用中 |
| **管理** | 状态管理 | config/model/output-style... | 会话状态管理与维护 | 按需调整会话参数 |
| | 安全管控 | permission mode/sandbox | 工具调用授权与运行环境隔离 | 文件和数据被误删、误改 |
| | 上下文管理 | 加载/压缩 | 上下文信息管理 | Token消耗 |

### 1.3 工具

工具是 Claude Code 的「手和脚」，LLM 推理出意图后，靠工具实际去读、写、改、执行本地环境。内置工具按用途可分为五类：

| 分类 | 能力 |
| --- | --- |
| **文件操作** | 读取文件、编辑代码、创建新文件、重命名和重新组织 |
| **搜索** | 按模式查找文件、使用正则表达式搜索内容、探索代码库 |
| **执行** | 运行 shell 命令、启动服务器、运行测试、使用 git |
| **网络** | 搜索网络、获取文档、查找错误消息 |
| **代码智能** | 编辑后查看类型错误和警告、跳转到定义、查找引用（需要代码智能插件） |

工具调用受 **权限模式（permission mode）** 管控，破坏性操作（删除、强推、覆盖等）会触发确认或直接拒绝。

### 1.4 接口特征对比

| 接口 | 本质 | 触发 | 用途 |
| --- | --- | --- | --- |
| 记忆 | 持久化知识 | 每次自动加载 | 规则与上下文 |
| 技能 | 把流程打包成按钮 | 用户输入 `/skill` 或任务匹配 | 规范化执行 |
| 子代理 | 委派子任务 | 主代理显式派发 | 隔离上下文、并行 |
| 钩子 | 事件触发器 | 工具/会话事件 | 自动化善后 |

## 三、知识设计

### 3.1 知识分类

| 维度 | 分类 | 子类 | 存放位置 | 加载方式 | 性质 | 内容举例 |
|---|---|---|---|---|---|---|
| **类型** | 规约知识 | 通用规则（范围规则） | `CLAUDE.md` | 每次会话启动即加载（常驻上下文，时刻生效） | 划定项目范围，指引大方向，限制模型行为，防止走偏 | 系统功能、技术栈、目录/功能结构、约束与限定 |
| | | 专属规则（领域规则） | `.claude/rules` | 系统按 Glob 模式条件触发 | 非业务，按技术分类组织，本质是对实现方式做规约 | 前端规约、后端规约、数据库规约等，规约包括：技术栈、命名规范、格式、组织结构、限定与约束、常用命令 |
| | 业务知识 |  | commands / skills / agents | 人为触发（按需） | 业务流程、领域模型、业务规则、命令组合 | 操作流程、计算过程、公式、输出模板 |
| **应用时机** | 编辑知识 |  | `.claude/rules` | 写入/修改文件时自动加载 | 编码规范、文件格式、目录约定 | 命名约定、缩进风格、注释规范、目录组织 |
| | 任务知识 |  | commands / skills / agents | 人为触发（按需） | 执行任务、命令、动作、计算时所需，通常为业务知识 | 操作流程、计算过程、公式、输出模板 |

### 3.2 知识设计原则

- **只放 AI 不知道的**：AI 已知的（通用语法、API 文档等）不写，只放项目特有信息
- **通用常驻**：通用规则放 `CLAUDE.md`，每次对话都要加载，常驻上下文
- **专属按需**：专属规则放 `.claude/rules`，系统按 Glob 模式条件触发
- **规约用记忆**：规则/约束用记忆机制（`CLAUDE.md` + `.claude/rules`）
- **业务用 skill**：业务知识用 skill（按需加载）
- **内容精简**：表达要精炼，少用虚词和修饰语，能用一行的不要写一段

## 四、使用进阶

从易到难、从被动到主动：口头告知 → 持久配置 → 代码编程

| 阶段 | 形式 | 作用 |
| --- | --- | --- |
| **口头告知** | 在对话中直接输入 | 每次对话都要输入相同的要求，用肉眼检查 |
| **持久配置** | CLAUDE.md / settings.json / commands / skills / agents / hooks | 用结构化配置管理规则、业务知识、权限、生命周期、自动化行为 |
| **代码编程** | Claude Agent SDK / API / Headless | 把 Agent 能力嵌入应用或脚本，由程序驱动 |

## 附录

- 凡事给 AI 看的文字，要按 AI 能清晰识别的方式去写，而不是按照人类阅读的习惯
